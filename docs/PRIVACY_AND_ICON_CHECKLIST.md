# プライバシー（5.1.2）とアイコン（2.3.8）対応チェックリスト

審査リジェクト対応のうち、まだの分の手順です。

---

## 1. プライバシー Guideline 5.1.2

**指摘**: 「トラッキングのため」に Device ID・Advertising Data を収集すると記載されているが、ATT（App Tracking Transparency）で許可を取っていない。

### アプリがトラッキングをしていない場合（推奨）

本アプリは広告のトラッキングやデータブローカーへの共有をしていない想定です。**App Store Connect のプライバシーラベルを修正**します。

1. **App Store Connect にログイン**
   - https://appstoreconnect.apple.com
   - **マイApp** → **宅建合格ロード** を選択

2. **アプリのプライバシーを開く**
   - 左サイドバー **「App Store」** の下 → **「アプリのプライバシー」** をクリック

3. **「データの収集」を確認・修正**
   - **「トラッキング」** または **「ユーザーのトラッキング」** に、Device ID・Advertising Data が「トラッキング目的」で並んでいないようにする。
   - **修正のしかた（どれかでOK）:**
     - **トラッキング目的のデータ収集を「なし」にする**  
       または
     - **該当データ（Device ID など）の「使用目的」から「トラッキング」のチェックを外す**  
       または
     - トラッキングに使っていないなら、そのデータタイプを**削除**する

4. **保存**
   - 変更後、必ず **保存** する。

※ **アプリのプライバシー** の編集には **アカウントホルダー** または **管理者** ロールが必要です。編集できない場合は、チームの管理者に依頼するか、リジェクトへの返信で「トラッキングは行っておらず、プライバシーラベルを修正します」と記載してください。

### アプリがトラッキングする場合

- トラッキングを開始する**前**に、`AppTrackingTransparency` で `requestTrackingAuthorization` を呼ぶ実装が必要です。
- 本アプリでは `Info.plist` に `NSUserTrackingUsageDescription` が設定済みです。実装は別途必要です。

---

## 2. アイコン Guideline 2.3.8

**指摘**: アプリアイコンがプレースホルダーのように見える。最終版のアイコンにする必要がある。

### 現状

- プロジェクトの **AppIcon** は `ios/App/App/Assets.xcassets/AppIcon.appiconset/` で管理されています。
- `Contents.json` では複数サイズの PNG が参照されていますが、**実際の PNG ファイルが入っていない**状態です（0 個）。

### 対応手順

#### ステップ1: アイコン画像を用意する

- **必須**: **1024×1024 px** の 1 枚（App Store 用。透過なし PNG 推奨）。
- それ以外のサイズは、次のどちらかで用意します。
  - **方法A**: 1024×1024 を元に、下のサイズにリサイズする（デザインツールやオンラインツール使用）。
  - **方法B**: Xcode で「Single Size」の AppIcon を使う（Xcode 14 以降）。  
    → AppIcon.appiconset で **「App Store iOS」用に 1024×1024 のみ** を設定し、他は空にできる場合があります（Xcode のバージョンにより挙動が異なります）。

#### ステップ2: 必要なサイズ一覧（方法A で全部入れる場合）

次のファイル名で、`ios/App/App/Assets.xcassets/AppIcon.appiconset/` に配置します。

| ファイル名 | ピクセルサイズ |
|------------|----------------|
| AppIcon-20x20@1x.png | 20×20 |
| AppIcon-20x20@2x.png | 40×40 |
| AppIcon-20x20@3x.png | 60×60 |
| AppIcon-29x29@1x.png | 29×29 |
| AppIcon-29x29@2x.png | 58×58 |
| AppIcon-29x29@3x.png | 87×87 |
| AppIcon-40x40@1x.png | 40×40 |
| AppIcon-40x40@2x.png | 80×80 |
| AppIcon-40x40@3x.png | 120×120 |
| AppIcon-60x60@2x.png | 120×120 |
| AppIcon-60x60@3x.png | 180×180 |
| AppIcon-76x76@1x.png | 76×76 |
| AppIcon-76x76@2x.png | 152×152 |
| AppIcon-83.5x83.5@2x.png | 167×167 |
| AppIcon-1024x1024@1x.png | **1024×1024**（必須） |

#### ステップ3: Xcode で確認

1. Xcode で `ios/App/App.xcworkspace` を開く。
2. 左のナビゲータで **App** → **Assets.xcassets** → **AppIcon** を開く。
3. 各スロットに画像が入っているか確認する。
4. 問題なければ **Product → Archive** で新しいビルドを作成し、App Store Connect にアップロードする。

### アイコン作成のヒント

- 1024×1024 は **角丸にしない**（Apple が自動でマスクをかけます）。
- デザインは全サイズで同じ意匠にし、ストアやホーム画面で同一アプリと分かるようにする。
- オンラインで「iOS app icon generator」などと検索すると、1024×1024 から各サイズを一括生成できるツールがあります。

---

## 3. 対応後の提出の流れ

1. **プライバシー** … App Store Connect 上で修正し保存。**新しいビルドは不要**です。
2. **アイコン** … 上記のとおり画像を入れてから、**新しいビルドを作成してアップロード**する必要があります。
3. 両方対応したうえで、**審査に提出**してください。

アイコンを変えた場合は、必ず **再度 Archive → アップロード** し、その新しいビルドを審査用に選択してください。
