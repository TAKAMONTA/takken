# 引き継ぎ書 2026-05-28

## 目的

App Store リリース済みの宅建学習アプリについて、アップデート作業として品質・安全性・保守性を上げる作業を進めていた。今回の主眼は、リリース前チェックの強化、問題データの重複/低品質データの削減、依存関係とビルド周辺の安全性改善。

## 現状の結論

- 問題文だけで見ていた重複判定は誤検知が多かったため、現在は「問題文 + 選択肢 + 正解」のフィンガープリントで判定するよう修正済み。
- アプリに実際に import されている問題ファイルだけを対象に、重複レポートの `file:line` を出せるようにした。
- 実データ上の完全重複は `0グループ / 0組` まで解消済み。
- 選択肢が全て同じでクイズとして成立しない問題も検出できるようにし、検出済み4件は削除済み。
- 問題数は最新チェックで合計 `407問`。
- ただし、最後に `check:question-quality` を `verify:release` へ組み込むところで作業停止。次回の最初にここを入れるのがよい。

## 追加・変更した主なチェック系ファイル

### 重複チェック

- `scripts/check/duplicates.ts`
  - 重複チェック本体。
  - `--report <path>` または `DUPLICATE_REPORT_PATH` で Markdown レポートを書き出せる。
  - 通常ログでは `MAX_DUPLICATE_GROUPS` で表示数を制限できる。
  - `buildQuestionSourceIndex(undefined, { activeOnly: true })` を使い、アプリで実際に使われる問題ファイルだけを対象化。

- `scripts/check/duplicate-utils.ts`
  - 重複判定の中核。
  - 判定基準は `question + options + correctAnswer`。
  - `explanation` は判定対象に含めていない。解説だけ違う同一問題は重複扱い。
  - 重複グループで残す問題は、メタ情報が豊かなものを優先する。
  - 優先スコアは `topic`, `frequency`, `grade`, `frequencyCount`, `explanation` 長さ, `relatedArticles`, `keyTerms`, `hints` など。

- `scripts/check/question-source-locator.ts`
  - 問題IDから `file:line` を引くユーティリティ。
  - `activeOnly: true` のとき、カテゴリの `index.ts` から import/export をたどり、未使用ファイルを除外する。
  - `_backup_copyrighted` や `utils` は除外。
  - 同じIDが別トピックで再利用されている場合に備え、問題文も併用して位置を絞る。

- `scripts/check/duplicate-report.ts`
  - Markdown形式の重複レポート整形。
  - `docs/duplicate-questions-report.md` を生成できるが、現在 `.gitignore` の `*-report.md` により Git 管理外になる。

### 品質チェック

- `scripts/check/question-quality-utils.ts`
  - 選択肢重複を検出するユーティリティ。
  - 全角スペースや連続空白は正規化して比較。

- `scripts/check/question-quality.ts`
  - 実データに対する品質チェック本体。
  - 現在は「選択肢重複」を検出。
  - `MAX_QUESTION_QUALITY_DETAILS` でログ表示数を制御できる。

### テスト

## 注意すべき設計判断

- 重複判定は現在「問題文 + 選択肢 + 正解」。問題文だけ同じ定型問題は削除しない。
- 解説だけ違う同一問題は重複扱い。残す側はメタ情報の豊富さで決める。
- `activeOnly` のソース追跡は static import/export を正規表現で追っている。複雑な動的 import は対象外。
- 問題品質チェックは現時点では選択肢重複のみ。今後追加するとよいチェック:
  - 正解 index が options 範囲外
  - options 数が2未満または4択問題で4未満
  - 空の question / explanation
  - category と配置ディレクトリの不一致
  - duplicate ID
